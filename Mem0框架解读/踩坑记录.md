# OpenMemory 使用手册

## 1. 使用 OpenMemory 官方面板

1. 打开浏览器访问 `https://app.openmemory.dev/dashboard`。  
2. 在面板中选择自己正在使用的 IDE（例如 Cursor、VS Code 等）。  
3. 按页面提示为对应的 IDE 安装并配置 OpenMemory MCP。
4. 安装完成后，在 IDE 中通过与智能助手对话的方式，就可以把记忆写入 OpenMemory。
5. 在 `https://app.openmemory.dev/dashboard` 面板中，可以查看、搜索和管理已经存储的记忆。


## 2. 使用 OpenMemory 本地部署

本次实验使用WSL+Docker Desktop部署OpenMemory，并使用Clash代理连接外部网络。

1. 进入WSL中的openmemory项目目录
2. 运行make env设置环境变量
3. 运行make build构建docker镜像
4. 运行make up启动容器


## 3. OpenMemory windows 本地部署踩坑记录(wsl+docker desktop)

### 坑 1：WSL 无法通过 Clash 代理访问网络

> 问题现象

    在wsl中运行make build下载docker镜像时报错，wsl无法连接windows电脑中的网络代理。

> 解决办法(可以配置docker镜像，但本次介绍连接外部科学上网，解决所有镜像问题)

    在 Windows 上使用 WSL2 开发时，WSL 本身和 Docker Desktop 需要分别配置代理才能访问外部网络。

> WSL 中配置代理

1. 找到 Windows 主机 IP

    WSL 通过宿主机网关访问外部网络，首先获取网关 IP：

    ```bash
    ip route | grep default | awk '{print $3}'
    ```
    输出类似 172.22.208.1，每台机器不同，不能照搬。

2. 确认 Clash 端口并开启 Allow LAN

    - 打开 Clash，在设置中确认**混合端口**（默认 `7890`）
    - 开启 **Allow LAN**（允许局域网），否则 WSL 的请求会被拒绝

3. 测试代理是否可用

    ```bash
    curl -v --proxy http://172.22.208.1:7890 https://www.google.com -I
    ```

    将其中的ip换成刚才获取的ip，有响应输出说明代理可用，再进行下一步。

4. 写入 ~/.bashrc 永久生效

    ```bash
    cat >> ~/.bashrc << 'EOF'

    # Clash 代理
    export WINDOWS_HOST=$(ip route | grep default | awk '{print $3}')
    export http_proxy=http://${WINDOWS_HOST}:7890
    export https_proxy=http://${WINDOWS_HOST}:7890
    export all_proxy=socks5://${WINDOWS_HOST}:7890
    EOF

    source ~/.bashrc
    ```
    > 用 $(ip route ...) 动态获取 IP，WSL 重启后 IP 变化也不需要手动更新。

    验证

    ```bash
    curl -I https://www.google.com
    ```

> Docker Desktop 中配置代理

`docker pull` 和 `docker build` 走的是 Docker Desktop 内部的代理配置，和 WSL 环境变量**相互独立**，需要单独设置。

1. 打开 Docker Desktop → Settings → Resources → Proxies，开启 Manual proxy configuration：

    | 字段 | 值 |
    |---|---|
    | Web Server (HTTP) | `http://172.22.208.1:7890` |
    | Secure Web Server (HTTPS) | `http://172.22.208.1:7890` |
    | Bypass proxy settings | `localhost,127.0.0.1` |

    通常换成在wsl中获取的ip，填好后点 **Apply & Restart**。

    验证一下，在wsl中运行以下命令，有响应输出说明代理可用：

    ```bash
    docker pull hello-world
    ```

2. 或配置 registry mirror（Docker Desktop -> Docker Engine）。
   可直接使用如下 JSON（按需替换可用镜像源）：

    ```json
    {
    "registry-mirrors": [
        "https://docker.1ms.run",
        "https://docker.xuanyuan.me"
    ]
    }
    ```


## 坑 2：容器启动报 `entrypoint.sh: no such file or directory`

> 问题现象

ui容器启动时报错，文件明明存在却找不到：

```
openmemory-ui-1  | exec /home/nextjs/entrypoint.sh: no such file or directory
```

> 根本原因

- 项目在 Windows 的 D 盘（`/mnt/d`），Git 在 Windows 上 clone 时默认将换行符转为 CRLF（`\r\n`）
- Linux 只认 LF（`\n`），CRLF 导致脚本解释器路径解析失败，表现为"文件不存在"

> 解决办法

用 `sed` 转换行尾符：
```bash
sed -i 's/\r//' ./path/to/entrypoint.sh
```

然后重新构建：`make build`


## 坑 3：`memory_client` 初始化失败

> 问题现象

- MCP 调用 `add_memories` 返回：`Memory system is currently unavailable. Please try again later.`

> 根本原因

- 后端在初始化 mem0 client 失败后进入降级模式（client 为 `None`）。
- 常见触发点：模型配置错误、embedding 接口不兼容、网络不通。

> 解决办法

1. 查看 `openmemory-mcp` 日志中 `Failed to initialize memory client`。
2. 修复配置后重试写入。
